# AWS 주요 관리형 서비스

## Cloudwatch

> 관리형 AWS 리소스 모니터링 서비스
> 

> AWS의 리소스 상태를 확인할 수 있는 다양한 metrics 제공 -> AWS의 어떤 서비스를 사용하든 cloudwatch에서 metric을 확인해야하는 경우 다수 발생.
> 

> 데이터 가시화 -> 대시보드 구성
> 

> SNS라는 알림 서비스와 통합하여 메트릭 수치에 대한 알람 받도록 구성 가능
> 

> 별도의 클라이언트 에이전트를 설치해서 사용자가 직접  지정한 커스텀 메트릭도 받을 수 있게 구성 가능
> 

## SNS (Simple Notification Service)

> 관리형 메시지 서비스
> 

> SNS를 통해 메시지를 발생시키는 생성자는 이메일이나 모바일 어플리케이션, 웹, 큐 등 다양한 형태의 메시지 전송 가능
> 

> 어플리케이션 간 메시징 가능
> 

> KMS와 통합하여 메시지 보안 구성 가능
> 

> 메시지 필터링 기능 제공
> 

> 메시지 생성자는 커뮤니케이션 채널인 Topic에 메시지 전송
> 

> Topic 구독자와 비동기식으로 통신
> 

> 구독자는 Kenesis나 sqs lambda http email 모바일 문자 메시지와 같은 엔드포인트를 사용해서 구독가능
> 

## Cloudtrail

> 관리형 이벤트 추적, 감사 도구
> 

> AWS 서비스가 수행하는 모든 작업들을 이벤트로그로 기록
> 

> 이 때 이벤트는 AWS 매니지먼트 콘솔, AWS CLI, SDK와 같이 API로 수행하는 모든 작업을 말함
> 

> cloudtrail을 통해 aws infra 전반에 걸친 계정 활동을 aws 관리 콘솔의 ui를 통해 쉽게 검색 가능 & 다운로드 받아 보관 가능
> 

> cloudtrail로 저장된 이벤트 활동 데이터는 데이터량이 크므로 원하는 데이터 검색 어려움 -> 그 경우, aws athena와 같은 분석 서비스를 사용해서 aws의 감사 서비스 분석 가능
>
